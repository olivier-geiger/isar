Pourquoi Isar ?

Une base de données
pensée pour Flutter

Isar est conçu du sol jusqu'au toit pour Dart/Flutter : zéro config, parfaitement typé, et des streams réactifs natifs.

⚡

Ultra-rapide

Moteur écrit en Rust. Aucun codec JSON, données lues directement depuis le disque avec zéro copie.

🔒

100 % typé

Généré à la compilation via build_runner. Zéro string SQL, zéro erreur de typo au runtime.

🔄

Réactif nativement

Chaque requête retourne un Stream. L'UI se reconstruit automatiquement à chaque changement.

🌐

Multi-plateformes

iOS, Android, macOS, Linux, Windows et le web (via IndexedDB) avec la même API.

🗂️

Relations & Index

IsarLinks pour les relations, index composites, full-text search intégré.

🔧

Inspecteur visuel

Isar Inspector dans le navigateur pour inspecter, filtrer et modifier vos données en live.

Étape 1

Installation

Ajoutez les dépendances dans pubspec.yaml et configurez le générateur de code.

pubspec.yaml yaml 
dependencies:
  flutter:
    sdk: flutter
  isar: ^3.1.0
  isar_flutter_libs: ^3.1.0
  path_provider: ^2.1.1

dev_dependencies:
  build_runner:
  isar_generator: ^3.1.0
Génération des fichiers shell 
# Installer les deps
flutter pub get

# Générer les adapters (une fois)
flutter pub run build_runner build

# Mode watch (développement)
flutter pub run build_runner watch \
  --delete-conflicting-outputs

main.dart — Ouverture de la base dart 
import 'package:isar/isar.dart';
import 'package:path_provider/path_provider.dart';
import 'models/task.dart';
import 'models/task.g.dart'; // généré automatiquement

late Isar isar;

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final dir = await getApplicationDocumentsDirectory();

  isar = await Isar.open(
    [TaskSchema],          // liste de tous vos schémas
    directory: dir.path,
    inspector: true,       // active l'inspecteur en debug
  );

  runApp(const MyApp());
}
Étape 2

Définir un Schéma

Annotez vos classes Dart avec @collection. Isar génère tout le reste automatiquement.

models/task.dart dart 
import 'package:isar/isar.dart';

part 'task.g.dart';   // fichier généré — ne pas modifier

@collection
class Task {
  Id id = Isar.autoIncrement;   // auto-incrément

  @Index(unique: true)           // index unique
  late String title;

  String? description;          // nullable

  bool isDone = false;

  @Index()                       // index pour tri rapide
  DateTime createdAt = DateTime.now();

  @enumerated                   // stocke l'enum en tant qu'entier
  Priority priority = Priority.normal;

  // Relation vers un User (voir section Relations)
  final assignee = IsarLink<User>();
}

enum Priority { low, normal, high }

// ── Autre collection ──────────────────────────────
@collection
class User {
  Id id = Isar.autoIncrement;

  @Index(type: IndexType.value)
  late String email;

  late String name;

  // Relation inverse (backlink)
  @Backlink(to: 'assignee')
  final tasks = IsarLinks<Task>();
}

💡 Après chaque modification d'un schéma, relancez build_runner build. Le fichier .g.dart sera régénéré automatiquement.
Étape 3

CRUD Complet

Toutes les opérations Create, Read, Update, Delete avec les patterns les plus courants.

Créer & sauvegarder des objets dart 
// ── Insérer un objet ────────────────────────────────
Future<void> createTask(String title) async {
  final task = Task()
    ..title = title
    ..priority = Priority.high;

  // writeTxn est OBLIGATOIRE pour toute écriture
  await isar.writeTxn(() async {
    await isar.tasks.put(task);   // put = insert ou update
  });
  // task.id est maintenant rempli automatiquement
}

// ── Insérer plusieurs objets d'un coup ──────────────
Future<void> bulkInsert(List<Task> tasks) async {
  await isar.writeTxn(() async {
    await isar.tasks.putAll(tasks);  // bulk insert
  });
}

// ── Avec relation (assigner un User) ────────────────
Future<void> createTaskWithUser(Task task, User user) async {
  await isar.writeTxn(() async {
    task.assignee.value = user;
    await isar.tasks.put(task);
    await task.assignee.save();   // sauvegarde la relation
  });
}
Lire & requêter des données dart 
// ── Par ID ──────────────────────────────────────────
Future<Task?> getById(int id) async {
  return isar.tasks.get(id);
}

// ── Tous les objets ─────────────────────────────────
Future<List<Task>> getAllTasks() async {
  return isar.tasks.where().findAll();
}

// ── Filtres typés ───────────────────────────────────
Future<List<Task>> getHighPriorityDone() async {
  return isar.tasks
    .filter()
    .priorityEqualTo(Priority.high)
    .and()
    .isDoneEqualTo(true)
    .findAll();
}

// ── Recherche textuelle (startsWith) ────────────────
Future<List<Task>> searchTasks(String query) async {
  return isar.tasks
    .where()
    .titleStartsWith(query)         // utilise l'index
    .findAll();
}

// ── Tri + pagination ────────────────────────────────
Future<List<Task>> getTasksPaged(int page) async {
  return isar.tasks
    .where()
    .sortByCreatedAtDesc()
    .offset(page * 20)
    .limit(20)
    .findAll();
}

// ── Compter sans charger ─────────────────────────────
Future<int> countPending() async {
  return isar.tasks.filter().isDoneEqualTo(false).count();
}
Mettre à jour des objets dart 
// ── Modifier et re-sauvegarder ──────────────────────
Future<void> toggleTask(int id) async {
  await isar.writeTxn(() async {
    final task = await isar.tasks.get(id);
    if (task != null) {
      task.isDone = !task.isDone;
      await isar.tasks.put(task);   // put = upsert
    }
  });
}

// ── Mise à jour partielle via property ──────────────
// (Disponible via isar_generator avec @ignore)
Future<void> updateTitle(int id, String newTitle) async {
  await isar.writeTxn(() async {
    final task = await isar.tasks.get(id);
    if (task != null) {
      task.title = newTitle;
      await isar.tasks.put(task);
    }
  });
}

// ── Mise à jour en masse ─────────────────────────────
Future<void> markAllDone() async {
  await isar.writeTxn(() async {
    final tasks = await isar.tasks.where().findAll();
    for (final t in tasks) {
      t.isDone = true;
    }
    await isar.tasks.putAll(tasks);
  });
}
Supprimer des objets dart 
// ── Par ID ──────────────────────────────────────────
Future<void> deleteTask(int id) async {
  await isar.writeTxn(() async {
    await isar.tasks.delete(id);
  });
}

// ── Plusieurs IDs à la fois ──────────────────────────
Future<void> deleteMany(List<int> ids) async {
  await isar.writeTxn(() async {
    await isar.tasks.deleteAll(ids);
  });
}

// ── Supprimer selon un filtre ────────────────────────
Future<void> deleteCompleted() async {
  await isar.writeTxn(() async {
    final done = await isar.tasks
      .filter().isDoneEqualTo(true).findAll();
    final ids = done.map((t) => t.id).toList();
    await isar.tasks.deleteAll(ids);
  });
}

// ── Vider toute une collection ───────────────────────
Future<void> clearAll() async {
  await isar.writeTxn(() async {
    await isar.tasks.clear();
  });
}

// ── Supprimer toute la base ──────────────────────────
Future<void> dropDatabase() async {
  await isar.writeTxn(() async {
    await isar.clear();  // toutes les collections
  });
}
Requêtes réactives — Stream + StreamBuilder dart 
// ── Stream sur toute une collection ─────────────────
Stream<List<Task>> watchAllTasks() {
  return isar.tasks
    .where()
    .sortByCreatedAtDesc()
    .watch(fireImmediately: true);
}

// ── Stream filtré ───────────────────────────────────
Stream<List<Task>> watchPending() {
  return isar.tasks
    .filter().isDoneEqualTo(false)
    .watch(fireImmediately: true);
}

// ── Utilisation dans un Widget ──────────────────────
class TaskList extends StatelessWidget {
  const TaskList({super.key});

  @override
  Widget build(BuildContext context) {
    return StreamBuilder<List<Task>>(
      stream: watchAllTasks(),
      builder: (context, snapshot) {
        if (!snapshot.hasData) return const CircularProgressIndicator();
        final tasks = snapshot.data!;
        return ListView.builder(
          itemCount: tasks.length,
          itemBuilder: (_, i) => TaskTile(task: tasks[i]),
        );
      },
    );
  }
}
Étape 4

Relations

Isar gère les relations 1-to-1 et 1-to-N via IsarLink et IsarLinks. Les relations sont lazy-loaded.

IsarLink — 1-to-1 dart 
// Définition dans le schéma
@collection
class Task {
  Id id = Isar.autoIncrement;
  late String title;

  // Lien vers un seul User
  final assignee = IsarLink<User>();
}

// Charger la relation
final task = await isar.tasks.get(1);
await task!.assignee.load();
final user = task.assignee.value;
IsarLinks — 1-to-N dart 
// Définition dans le schéma
@collection
class Project {
  Id id = Isar.autoIncrement;
  late String name;

  // Lien vers plusieurs Tasks
  final tasks = IsarLinks<Task>();
}

// Charger toutes les relations
final project = await isar.projects.get(1);
await project!.tasks.load();
final list = project.tasks.toList();

Sauvegarder & supprimer des relations dart 
// ── Ajouter des éléments à IsarLinks ────────────────
await isar.writeTxn(() async {
  project.tasks.addAll([task1, task2]);
  await project.tasks.save();
});

// ── Retirer un élément ──────────────────────────────
await isar.writeTxn(() async {
  project.tasks.remove(task1);
  await project.tasks.save();
});

// ── Backlink (relation inverse) ──────────────────────
@collection
class User {
  Id id = Isar.autoIncrement;
  late String name;

  // Accès à toutes les tâches assignées à ce user
  @Backlink(to: 'assignee')
  final assignedTasks = IsarLinks<Task>();
}

// Utilisation du backlink (lecture seulement)
await user.assignedTasks.load();
print(user.assignedTasks.length);
Étape 5

Migrations de Schéma

Isar gère automatiquement les migrations de schéma. Voici comment gérer les cas complexes et les données existantes.

⚠️ Isar migre automatiquement les schémas (ajout/suppression de champs). Mais si vous renommez un champ ou changez son type, vous devez gérer la migration manuellement via migration.
1

Incrémentez le schemaVersion

À chaque changement de schéma incompatible, incrémentez le numéro de version.

Isar.open avec versiondart
isar = await Isar.open(
  [TaskSchema, UserSchema],
  directory: dir.path,
  name: 'myApp',
  schemaVersion: 2,       // ← incrémentez ici
  migration: IsarMigration(),
);
2

Implémentez IsarMigration

Gérez la logique de transformation des données existantes entre les versions.

migration.dartdart
class IsarMigration extends IsarMigration {
  @override
  Future<void> migrate(
    Isar isar,
    int oldVersion,
    int newVersion,
  ) async {
    if (oldVersion <= 1) {
      // v1 → v2 : renommer 'name' en 'title'
      await migrateV1ToV2(isar);
    }
    if (oldVersion <= 2) {
      // v2 → v3 : ajouter le champ priority avec valeur par défaut
      await migrateV2ToV3(isar);
    }
  }

  Future<void> migrateV1ToV2(Isar isar) async {
    await isar.writeTxn(() async {
      final tasks = await isar.tasks.where().findAll();
      for (final task in tasks) {
        // Migrer les données
        task.priority = Priority.normal;
      }
      await isar.tasks.putAll(tasks);
    });
  }

  Future<void> migrateV2ToV3(Isar isar) async {
    // Isar ajoute les nouveaux champs automatiquement
    // (avec leur valeur par défaut Dart)
    // Ici on peut faire de la logique custom si besoin
  }
}
3

Ce qu'Isar gère automatiquement

Pas besoin de migration pour les cas simples.

✅

Ajout de champ

Les nouveaux champs reçoivent leur valeur Dart par défaut pour les objets existants.

✅

Suppression de champ

Les données du champ supprimé sont ignorées, aucune perte d'autres données.

⚠️

Renommage

Isar voit ça comme un ajout + suppression. Migration manuelle nécessaire pour transférer les valeurs.

⚠️

Changement de type

Migration manuelle obligatoire. Isar ne peut pas convertir les types automatiquement.
